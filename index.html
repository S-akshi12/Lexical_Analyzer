Compiler Lexical Analysis: Code to Tokens

Compiler Lexical Analysis

Character Stream

The input to the lexical analyzer is usually a stream of characters, such as the source code of a program written in a programming language. This stream can come from a variety of sources, including files, user input, or network communication.

Tokenization

The lexical analyzer reads characters from the input stream and groups them into meaningful units called tokens. Tokens are the smallest units in a programming language, such as keywords, identifiers, literals (numbers or strings), operators, and punctuation symbols.

Lexical Analysis

During the process of tokenization, the lexical analyzer also performs lexical analysis to identify and discard any whitespace characters (such as spaces and tabs) and comments, which are typically not relevant to the subsequent phases of compilation.

Token Output

As tokens are recognized, the lexical analyzer typically generates output in the form of a token stream. Each token in the stream contains information about the type of token (e.g., keyword, identifier, number) and, in some cases, the actual lexeme (the portion of the input text that corresponds to the token).
